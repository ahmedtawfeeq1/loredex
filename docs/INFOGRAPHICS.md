# Infographics

This is the public visual walkthrough for loredex: what the problem is, how the system works,
why it matters, and how someone can start fast.

## Read order

1. Start with the problem and the big idea.
2. Move into architecture, classification, and vault structure.
3. Finish with onboarding, curation, MCP, and cross-agent usage.

## 1. Problem and concept

### 01. Cover

The one-line framing: loredex gives your agents a real memory system instead of a pile of markdown.

![Loredex cover](assets/01-cover.png)

### 02. The problem

Shows the core pain clearly: agents create useful notes, but those notes become fragmented, forgotten, and hard to reuse.

![The problem Loredex solves](assets/02-the-problem.png)
